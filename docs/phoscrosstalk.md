### Variant 1: The "Hybrid" Run (Recommended)

**Use Case:** You want to use the curated databases for PTM crosstalk, but your **own custom file** for Kinase-Substrate interactions.
*This is the command derived in our previous step.*

```bash
phoscrosstalk \
  --data data_timeseries/filtered_input1.csv \
  --ptm-intra data_curated/processed/ptm_intra.db \
  --ptm-inter data_curated/processed/ptm_inter.db \
  --kinase-tsv data_interactions/kinase_sites.tsv \
  --unified-graph-pkl data_curated/processed/unified_kinase_graph.gpickle \
  --outdir network_random_thesis \
  --cores 95 \
  --mechanism rand \
  --gen 300 \
  --run-steadystate \
  --run-knockouts \
  --run-sensitivity

```

### Variant 2: The "Fully Curated" Run

**Use Case:** You don't have a custom kinase file (or trust the public databases more). This uses the **KEA/PhosphoSitePlus mapping** generated by your curator script instead of your `input2.csv`.

**Change:** Replaced `--kinase-tsv` with `--kea-ks-table`.

```bash
phoscrosstalk \
  --data data_timeseries/filtered_input1.csv \
  --ptm-intra data_curated/processed/ptm_intra.db \
  --ptm-inter data_curated/processed/ptm_inter.db \
  --kea-ks-table data_curated/processed/ks_psite_table.tsv \
  --unified-graph-pkl data_curated/processed/unified_kinase_graph.gpickle \
  --outdir network_fully_curated \
  --cores 95 \
  --mechanism rand \
  --gen 300 \
  --run-steadystate --run-knockouts --run-sensitivity

```

### Variant 3: The "Mechanism Comparison" Runs

**Use Case:** You want to test if the phosphorylation is **Distributive** (independent sites) or **Sequential** (ordered) instead of Random/Cooperative.

**Change:** `--mechanism` flag.

**Distributive Model:**

```bash
phoscrosstalk \
  ... (same inputs) ... \
  --outdir network_distributive \
  --mechanism dist \
  --gen 300

```

**Sequential Model:**

```bash
phoscrosstalk \
  ... (same inputs) ... \
  --outdir network_sequential \
  --mechanism seq \
  --gen 300

```

### Variant 4: The "Hyperparameter Tuning" Run

**Use Case:** Before running the full 300 generations, you want the model to automatically find the best values for `length-scale` (local decay) and `lambda-net` (regularization strength).

**Change:** Added `--tune`. Note that `--tune` usually runs before the main fitting loop.

```bash
phoscrosstalk \
  --data data_timeseries/filtered_input1.csv \
  --ptm-intra data_curated/processed/ptm_intra.db \
  --ptm-inter data_curated/processed/ptm_inter.db \
  --kinase-tsv data_interactions/kinase_sites.tsv \
  --unified-graph-pkl data_curated/processed/unified_kinase_graph.gpickle \
  --outdir network_tuning_run \
  --cores 95 \
  --mechanism rand \
  --tune \
  --gen 300

```
